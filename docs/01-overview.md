# 01 项目总览

## 1.1 定位

MealPing 是一个轻量级扫码点单系统，目标是用最少的功能跑通 **「扫码 → 点菜 → 提交 → 入库 → 后台看单 → 通知店主」** 这条核心链路。

## 1.2 当前版本范围

| 阶段 | 状态 | 说明 |
| --- | --- | --- |
| Phase 1：点单闭环 | ✅ 已实现 | 菜单、购物车、下单、后台订单管理 |
| Phase 2：公众号通知 | ✅ 已实现 | 微信公众号测试号模板消息 + 通知日志 |
| Phase 3：菜单管理 | ✅ 已实现 | 分类、菜品 CRUD、上下架 |
| Phase 4：体验优化 | ⚠️ 部分实现 | SSE 实时推送 + 提示音；二维码生成留作 TODO |

## 1.3 角色

| 角色 | 入口 | 鉴权 |
| --- | --- | --- |
| 顾客 | `/menu?table=xxx` | 免登录 |
| 店主/管理员 | `/admin/*` | 共享密码 + JWT |

## 1.4 非目标范围

不做：配送、在线支付、会员、优惠券、多门店、骑手端、复杂收银、库存。

## 1.5 关键设计原则

1. **金额以后端为准** — `OrderItem.unitPrice` 由服务端在事务中从数据库读取后写入，前端传入的价格只用于本地显示。
2. **事务一致性** — 订单与订单明细在同一 Prisma 事务中创建。
3. **通知不阻塞下单** — `notifyOrderCreated` 异步触发，失败仅写入 `NotifyLog` 并标记 `notifyStatus=failed`。
4. **简单胜过强大** — 单管理员密码登录，桌号通过 URL 参数传递，不引入复杂权限体系。
